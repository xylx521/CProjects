## 一、指针基础

### 核心概念

内存就像公寓楼，每个房间有门牌号（地址），住着数据（值）。**指针就是一张写着门牌号的纸。**

```c
int a = 42;
int *p = &a;    // & 取地址，得到门牌号，存入 p
int val = *p;   // * 解引用，拿着纸去房间取值
*p = 100;       // * 解引用，拿着纸去房间改值
```

| 符号 | 位置 | 含义 |
|------|------|------|
| `&` | 变量前 | 取地址 |
| `*` | 声明时 | 声明这是一个指针 |
| `*` | 使用时 | 解引用，去地址取值或写值 |

---

### 指针运算

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;

*(p + 0)   // 10，arr[0]
*(p + 1)   // 20，arr[1]，跳过 4 字节
*(p + 2)   // 30，arr[2]，跳过 8 字节
```

> `arr[i]` 本质是 `*(arr + i)`，下标是语法糖。指针加 1 = 跳过一个元素大小，不是跳过 1 字节。

---

### 野指针 vs NULL 指针

```c
int *p;          // 野指针：未初始化，指向随机地址
*p = 42;         // 可能悄悄破坏其他数据，不立即崩溃，极难排查

int *p = NULL;   // NULL 指针：明确指向地址 0
*p = 42;         // 立即触发段错误/HardFault，崩溃暴露问题反而更安全
```

> 野指针比 NULL 指针更危险——NULL 崩溃立即暴露，野指针可能悄悄破坏数据很久才发现。

## 二、指针的高级用法

### 常量指针 vs 指针常量

口诀：**`const` 在 `*` 左边管值，在 `*` 右边管方向。**

```c
const int *p;        // 指向常量的指针（指针常量）
                     // 可以换门牌号，不能改房间内容
int * const p;       // 常量指针
                     // 不能换门牌号，可以改房间内容
const int * const p; // 两者都不能改
```

用公寓楼理解：
- `const int *p`：这张纸可以换，但拿着纸找到的房间不能动
- `int * const p`：这张纸不能换，但房间里的东西随便改

---

### 指针数组 vs 数组指针

口诀：**从变量名开始，先往右读，再往左读。**

```c
int *p[5];     // 指针数组：p 是数组，里面装了 5 个 int* 指针
int (*p)[5];   // 数组指针：p 是指针，指向一个 int[5] 数组
```

---

### 函数指针 vs 指针函数

```c
// 指针函数：返回值是指针的普通函数
int* get_data(void) {
    static int val = 42;
    return &val;         // 返回一个 int*
}

// 函数指针：指向函数的指针
int add(int a, int b) { return a + b; }

typedef int (*CalcFunc)(int, int);   // 用 typedef 简化
CalcFunc fp = add;
fp(3, 4);    // 等价于 add(3, 4)
```

| | 函数指针 | 指针函数 |
|--|---------|---------|
| 本质 | 是一个指针 | 是一个函数 |
| 用途 | 回调、状态机、HAL | 返回动态数据地址 |
| 声明 | `int (*fp)(int, int)` | `int* func(void)` |

**函数指针三大用途：**

```c
// 1. 回调函数（FreeRTOS 任务传入方式）
xTaskCreate(vTaskFunction, "Task1", 128, NULL, 1, NULL);

// 2. 状态机（枚举 + 函数指针数组）
typedef void (*StateHandler)(void);
StateHandler handlers[STATE_MAX] = {do_idle, do_collect, do_transmit};
handlers[current_state]();

// 3. 硬件抽象层（切换传感器驱动只换指针）
typedef float (*ReadTemp)(void);
ReadTemp read = use_dht11 ? dht11_read : ds18b20_read;
```

---

### 嵌入式寄存器访问

所有 STM32 外设寄存器本质都是指针操作：

```c
// 向 GPIO 寄存器地址写值，控制引脚电平
*(volatile uint32_t *)(0x40011010) = 0x00000020;

// HAL 库底层等价写法
GPIOA->ODR |= (1U << 5);
```

## 三、内存模型

### 四个区域

```
高地址 ┌─────────────────┐
       │     栈 Stack     │  向下增长 ↓
       │                 │
       │     堆 Heap      │  向上增长 ↑
       ├─────────────────┤
       │   全局 / 静态区  │
       ├─────────────────┤
低地址 │     代码区       │  只读，存机器指令
       └─────────────────┘
```

| 区域 | 分配方式 | 释放方式 | STM32 位置 | 特点 |
|------|---------|---------|-----------|------|
| 代码区 | 编译时 | 不释放 | Flash | 只读 |
| 全局/静态区 | 启动时 | 程序结束 | RAM | 自动清零 |
| 栈 | 函数调用 | 函数返回 | RAM | 快，但空间小 |
| 堆 | `malloc` | `free` | RAM | 灵活，需手动管理 |

---

### `static` 关键字

```c
// 用法1：静态局部变量
// 长得像局部变量，实际住在全局区，生命周期 = 整个程序
void count_calls(void) {
    static int count = 0;
    count++;              // 每次调用都保留上次的值
    printf("%d\n", count);
}
count_calls();  // 1
count_calls();  // 2
count_calls();  // 3

// 用法2：静态全局变量/函数
// 限制作用域，只在本文件可见，防止命名冲突
static int module_state = 0;       // 其他文件看不到
static void helper(void) { ... }   // 其他文件看不到
```

---

### `malloc` 与 `free`

```c
int *arr = malloc(5 * sizeof(int));  // 在堆上申请 20 字节

if (arr == NULL) {     // 必须检查！申请失败返回 NULL
    return -1;
}

// 使用...

free(arr);             // 用完必须释放
arr = NULL;            // 防止悬空指针
```

---

### 内存泄漏

```c
void leak_demo(void) {
    int *p = malloc(100);
    // 忘记 free(p)
}   // p 随栈消失，堆上那 100 字节永远无法回收
```

**嵌入式 `malloc` 的两大坑：**

| 问题 | 原因 | 后果 |
|------|------|------|
| 内存泄漏 | 忘记 `free`，指针随栈消失 | 堆越来越小，最终耗尽 |
| 内存碎片 | 反复申请释放，堆不连续 | 大块申请失败，即使总量够 |

> 嵌入式替代方案：**静态内存池**——提前分配大数组，自己管理。FreeRTOS 就是这么做的。

---

### STM32 内存布局（真实地址）

```
0x20000000  ┌─────────────────┐  RAM 起始（20KB）
            │   全局 / 静态区  │  低地址
            ├─────────────────┤
            │      堆          │  ↑ 向上增长
            │                 │
            │      栈          │  ↓ 向下增长
0x20004FFF  └─────────────────┘  RAM 结束
```

### ⚠️ 两大内存坑

**坑1：栈溢出**

```c
void bad(void) {
    char buffer[4096];  // STM32 栈默认只有 512B~几KB
}                       // 溢出 → 覆盖相邻内存 → HardFault
                        // 危险：不一定立即崩溃，可能隔很久才出现诡异 bug
```

规则：大数组放全局区，不放局部变量。

**坑2：悬空指针**

```c
int *p = malloc(4);
free(p);
*p = 42;    // p 指向已释放内存 → 未定义行为
// 修法：free 后立即 p = NULL
```

## 四、位操作

### 为什么必须会

STM32 寄存器是 32 位的，每个 bit 控制一个硬件功能。直接 `=` 赋值会清掉其他位，破坏其他引脚配置，**必须用位操作精确控制单个 bit。**

---

### 三个核心公式

```c
reg |=  (1U << n);    // 置1：bit n 置位，其他位不变
reg &= ~(1U << n);    // 置0：bit n 清位，其他位不变
reg &   (1U << n);    // 检测：bit n 是否为 1
reg ^=  (1U << n);    // 翻转：bit n 取反，其他位不变
```

**拆解 `&= ~(1U << n)` 的每一步：**

```
以 n=5 为例：
1U << 5         = 0b00100000   把第5位置1
~(1U << 5)      = 0b11011111   取反，第5位变0其余变1
reg &= 0b11011111              按位与，reg 第5位清0，其余保持不变
```

---

### 检测多个 bit 同时为1

```c
// 检测 bit3 和 bit6 同时为1
uint8_t mask = (1U << 3) | (1U << 6);   // 0b01001000
if ((reg & mask) == mask) {
    printf("both set\n");
}
```

---

### 实用宏定义

```c
#define BIT(n)        (1U << (n))
#define SET_BIT(r, n) ((r) |=  BIT(n))
#define CLR_BIT(r, n) ((r) &= ~BIT(n))
#define TGL_BIT(r, n) ((r) ^=  BIT(n))
#define GET_BIT(r, n) (((r) >> (n)) & 1U)
```

---

### STM32 实战

```c
RCC->APB2ENR |=  (1U << 2);     // 使能 GPIOA 时钟
GPIOA->CRL   &= ~(0xF << 20);   // 清除 PA5 配置位
GPIOA->CRL   |=  (0x3 << 20);   // 设为推挽输出
GPIOA->ODR   |=  (1U << 5);     // PA5 高电平
GPIOA->ODR   &= ~(1U << 5);     // PA5 低电平
GPIOA->ODR   ^=  (1U << 5);     // PA5 翻转
```

---

### ⚠️ 易错点

**易错1：检测用 `&`，不是 `|=`**

```c
// ❌ 错误：|= 是置位操作，条件永远成立
if (reg |= (1 << 3)) { ... }

// ✅ 正确
if (reg & (1 << 3)) { ... }
```

**易错2：移位用 `1U` 不用 `1`**

```c
1  << 31   // ❌ 有符号 int 移到符号位，未定义行为
1U << 31   // ✅ 无符号，安全
```

## 五、宏定义 `#define`

### 常量宏 vs `const`

```c
#define MAX_SIZE  128      // 纯文本替换，无类型，无作用域
const int max_size = 128;  // 有类型，有作用域，调试器可见
```

> 嵌入式常量优先用 `const` 或 `enum`，宏只用于条件编译和特殊场合。

---

### 函数式宏两个括号规则

```c
// ❌ 危险写法
#define DOUBLE(x)  x + x

// ✅ 正确写法：参数加括号 + 整体加括号
#define DOUBLE(x)  ((x) + (x))
```

---

### 条件编译

```c
#define DEBUG

#ifdef DEBUG
    #define LOG(fmt, ...) \
        printf("[%s:%d] " fmt "\n", __func__, __LINE__, ##__VA_ARGS__)
#else
    #define LOG(fmt, ...)   // 发布版本，零开销
#endif

// 跨平台
#if MCU_TYPE == 32
    typedef uint32_t reg_t;
#elif MCU_TYPE == 16
    typedef uint16_t reg_t;
#else
    #error "不支持的 MCU 类型"
#endif
```

---

### 头文件保护

```c
// 方式一：Header Guard（可移植）
#ifndef SENSOR_H
#define SENSOR_H
// ... 内容 ...
#endif

// 方式二：pragma once（简洁，CubeIDE 默认）
#pragma once
// ... 内容 ...
```

---

### `static inline` 替代函数式宏

```c
// 宏：无类型检查，有副作用风险
#define SQUARE(x)  ((x) * (x))

// static inline：有类型检查，无副作用，可调试
static inline int square(int x) { return x * x; }
```

---

### ⚠️ 易错点

**易错1：函数式宏漏括号**

```c
#define DOUBLE(x)  (x) + (x)    // ❌ 整体没括号
DOUBLE(3) * 2                   // → 3 + 3*2 = 9，不是 12
```

**易错2：宏传入有副作用的表达式**

```c
SQUARE(i++)   // ❌ 展开：((i++) * (i++))，i 自增两次
square(i++)   // ✅ static inline，i++ 只执行一次
```

**易错3：`#ifdef` 只检查是否定义，不看值**

```c
#define DEBUG 0

#ifdef DEBUG   // ✅ 会进来，因为 DEBUG 被定义了（即使值是0）
#if DEBUG      // ❌ 不进来，因为值是 0
```
